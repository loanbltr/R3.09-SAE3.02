ServerMaster module¶

Cluster de calcul (côté serveur maître)

Cette application met en œuvre un serveur maître utilisant PyQt5 pour l’interface utilisateur graphique (GUI) et des sockets pour la communication avec plusieurs clients et plusieurs serveurs maîtres. Le serveur reçoit des messages de clients, les traite et les envoie à des serveurs esclaves. Les serveurs esclaves renvoient les résultats au serveur maître qui les renvoie aux clients. Le serveur maître peut également communiquer avec des serveurs esclaves pour obtenir des informations sur les performances du CPU, de la RAM et les langages de programmation disponibles.

Classes:

  • ServerMaster: classe chargée de gérer la communication entre le client et le serveur.

  • ServerMasterGUI: classe basée sur QWidget chargée de gérer la communication entre le client et le serveur esclave.

  • SecondaryWindow: classe basée sur QWidget chargée de gérer l’affichage des informations du serveur esclave.

Utilisation :

  1. Lancer l’application.

  2. Définissez si vous souhaitez utiliser l’adresse IP locale ou une adresse IP spécifique.

  3. Définissez les ports pour le client, le serveur esclave et le serveur esclave CPU.

  4. Cliquez sur le bouton « Start Server » pour démarrer le serveur.

  5. L’utilisateur a la possibilité de visualiser les informations du serveur esclave (OPTIONNEL).
